## Salesforce MCP Server — Agent Guidelines
You have access to a Salesforce MCP server with 3 tools. All tools authenticate
via the caller's JWT identity and enforce Salesforce security policies per-user.
### Recommended Tool Workflow
1. **describe_salesforce_object** → Call FIRST to discover an object's available
   fields, types, and relationships before writing any SOQL.
2. **query_salesforce** → Execute SOQL queries. Always check the `success` field
   in the response. Fields may be silently removed; never hallucinate missing data.
3. **upload_account_plan** → Create or update account plans (upsert by Account +
   Plan Year). Validate required fields before calling.
### Key Constraints
- **SecuredSOQL is NOT standard SOQL.** Only SELECT, WHERE, ORDER BY, LIMIT,
  OFFSET are supported. No subqueries, no aggregate functions.
- **PII fields in WHERE/ORDER BY will BLOCK the query** (403 "Inference attack
  detected"). If this happens, remove the restricted field and retry with only
  SELECT.
- **Silently filtered fields:** Compare requested vs returned fields. If a field
  is missing from the response, tell the user it was filtered by security policy.
  Never invent values.
- **Row-level security:** Record count may be less than LIMIT. Do not assume
  more records exist.
- **Object allowlist:** Not all Salesforce objects are queryable. If you get
  "not permitted for querying", inform the user.
